# Warpkeep Alpha 0.3.4 — Realm Quality Follow-through

**Status: Pages-only Alpha 0.3.4 release. Its protected-main Pages workflow and
exact public build stamp are the deployment record. No Worker, Durable Object,

Protected-main deployment: [089430e](https://github.com/ael-dev3/Warpkeep/commit/089430ecec83b72756104b33632673bdc6c2d8f1),

Alpha 0.3.4 follows through on the Genesis realm presentation repair without
changing player authority. It keeps the canonical protocol-3 Genesis 001 world,
admission boundary, Terms gate, browser-bound Farcaster sign-in, and Marks
boundary intact.

## Release scope

The Pages-only release scope contains frontend presentation, local regression
coverage, documentation, and build-exclusion hardening. It does not authorize
or perform:

- a Cloudflare Worker deployment or Durable Object migration;
- a SpacetimeDB module/schema publication;
- a QA-observer enablement, registration, or credential change;
- profile refresh/backfill, admission, world, castle, wallet, Marks, scan, or
  burn mutation.

The repository includes disabled, machine-bound QA observer code for controlled
local testing. It is explicitly outside the Pages-only release scope. The
production build verifier rejects its endpoint, procedure, configuration, local
fixture, and broker markers from player assets.

## Release improvements

- The title route uses only the integrity-verified 3D title model; retired HTML,
  SVG, loader, and procedural wordmark paths no longer duplicate it.
- Castle identity labels use active-LOD projected hulls and measured bounds to
  remain roof-attached, readable, and clear of castle silhouettes, UI regions,
  and one another across dense clusters.
- The new high, balanced, and compact Hegemony Main Castle GLBs replace the
  earlier Frontier Keep derivatives while preserving shared instancing,
  integrity pinning, and production LOD budgets.
- The compact Hegemony castle regression decodes the exact integrity-pinned GLB
  through the production Meshopt loader, creates the shared prefab/instance
  layer, and proves a genuine WebGL-canvas pointer sequence opens the intended
  castle inspector before terrain fallback can apply.
- The local rendered-WebGL matrix now includes the full player presentation at
  390×844, a tablet player inspector, and short-landscape player Explore,
  including compact Menu/Home/Explore chrome, touch targets, map/model parity,
  and no horizontal overflow. The existing desktop, observer, reduced-quality,
  cluster, and explorer cases are retained.
- A local-only, fail-closed source-versus-runtime WebGL lane compares the
  authorized source keep with the high, balanced, and compact runtime keeps
  through the shared production preparation path. It records bounded aggregate
  fidelity metrics only; source bytes, screenshots, pixels, paths, hashes,
  identities, and browser logs remain private and are not bundled.
